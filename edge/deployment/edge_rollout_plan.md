# Edge Rollout Plan

## Phase 1: Internal Validation
- Validate conversion scripts and runtime wrappers.
- Establish baseline latency and probability deviation limits.

## Phase 2: Beta Devices
- Roll out to limited Android/iOS cohort.
- Enable telemetry for failures and fallback frequency.

## Phase 3: Extension Pilot
- Release unlisted extension build to pilot users.
- Tune scan limits and overlay UX from feedback.

## Phase 4: Public Release
- Publish mobile binaries and extension listing.
- Monitor sync reliability, model drift, and abuse signals.

## Phase 5: Continuous Updates
- Versioned model rollout from Module 6 MLOps registry.
- Controlled rollback strategy for problematic edge artifacts.
